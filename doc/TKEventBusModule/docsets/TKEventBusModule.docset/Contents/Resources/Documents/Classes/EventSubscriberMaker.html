  • subscribe(_:complation:)

    合并订阅多个事件(融合订阅)

    将多个事件合并为一个可事件回调，当满足所有的事件时，才会触发回调

  • Example:

    self.bus.subscribe([.login,.logout]) { (events) in
    /// do some thing
}

    • Note

    合并多个事件，及将当前给定的 events 多个事件合并为一个事件回调， 只有满足所有的事件时，才会触发回调

Pause/ restore
  • pauseSubscribe(on:)

    暂停订阅

    此方法推荐在 viewWillDisappear 等方法中调用，及当前暂时不想接收此事件的的地方调用

    Precondition

    此方法需要在已经订阅事件后才有效，未订阅事件调用无效

    Note

    此方法不会取消事件监听，只是暂停

    Precondition

    此方法 暂时不支持 联合事件

  • unsubscribe(on:fusion:)

    取消订阅融合类型事件

  • Example:

        self.bus.ubsubscribe(on: .login, fusion: true)

    • Note

    默认情况下, fusion是false的， 取消订阅当前事件，不会影响到联合订阅部分，如果 fusion是true,当联合事件中包含此事件时，将不会触发事件回调，及也取消了联合事件订阅

    Declaration

    Swift

    @discardableResult
public func unsubscribe(on name: TKEvent.Name, fusion: Bool = false) -> Self

    Parameters

    name

    事件名称
    fusion

    是否取消包含此事件的融合事件 默认情况下，只会去除融合事件中关于此事件的内容， true: 会整个删除包含此事件的融合事件订阅 default: false
